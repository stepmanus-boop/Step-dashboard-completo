CORREÇÃO COMPLETA APLICADA - LOGO PRIO + IMAGENS INDIVIDUAIS POR UNIDADE

O que foi alterado no projeto completo:

1. ADMINISTRAÇÃO > USUÁRIOS E ALERTAS SETORIAIS
   - Adicionados campos:
     - Cliente vinculado
     - Nome exibido no portal
     - Logo do cliente
     - Upload da logo do cliente
     - Imagem padrão das unidades
     - Upload da imagem padrão
     - Imagens individuais das unidades/plataformas
   - Cada unidade agora pode receber sua própria imagem.
   - Ex.: FORTE, FRADE, BRAVO, FPSO FRADE podem ter fotos diferentes.

2. PORTAL / CARDS
   - A logo do cliente usa client_logo_url.
   - A imagem padrão continua em client_platform_image_url apenas como fallback.
   - Os cards de unidade buscam client_unit_images por client_name + unit_name.
   - Se uma unidade não tiver imagem, usa a imagem padrão do cliente.
   - A logo do cliente também pode substituir a logo STEP no topo quando o usuário logado tiver logo cadastrada.

3. NETLIFY FUNCTIONS
   - Nova função /api/admin-client-images para upload/salvamento de imagens.
   - Nova função /api/client-images para leitura das imagens por usuários logados.
   - admin-users agora salva e retorna:
     - clientName
     - portalDisplayName
     - clientLogoUrl
     - clientPlatformImageUrl
   - auth-login e auth-me agora enviam os dados da logo/cliente na sessão.
   - _supabase agora entende os novos campos e a tabela client_unit_images.

4. SUPABASE
   Rode o arquivo abaixo no SQL Editor:
   supabase/SQL-CLIENT-LOGO-UNIT-IMAGES.sql

   Ele cria:
   - users.client_name
   - users.portal_display_name
   - users.client_logo_url
   - users.client_platform_image_url
   - client_unit_images
   - buckets client-logos e client-unit-images
   - policies de storage
   - reload schema para evitar PGRST204

5. CACHE/PWA
   - Atualizado o nome do cache do service worker para forçar atualização dos arquivos.

PASSO A PASSO:
1. Rode o SQL em supabase/SQL-CLIENT-LOGO-UNIT-IMAGES.sql.
2. Suba este ZIP completo no Netlify/GitHub.
3. Limpe cache do navegador ou reinstale o PWA.
4. Entre como admin.
5. Edite o usuário PRIO.
6. Preencha Cliente vinculado = PRIO.
7. Importe a logo da PRIO.
8. Em "Imagens das unidades / plataformas", importe a foto de cada unidade.
9. Salve.
