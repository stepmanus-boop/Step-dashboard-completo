CONFIGURAÇÃO DO GITHUB

1. Antes de publicar o projeto, abra a pasta config
2. Edite o arquivo github-sync.json
3. Preencha:
   - token
   - repo
   - branch

Exemplo:
{
  "token": "SEU_TOKEN_AQUI",
  "repo": "stepmanus-boop/Step-dashboard",
  "branch": "main"
}

IMPORTANTE
- O campo de token não aparece mais no painel admin
- O botão "Subir pro GitHub" continua no admin
- Depois do deploy, o sistema usa esse arquivo para sincronizar usuários e alertas
- Se quiser trocar o token depois, altere o arquivo config/github-sync.json e publique novamente
