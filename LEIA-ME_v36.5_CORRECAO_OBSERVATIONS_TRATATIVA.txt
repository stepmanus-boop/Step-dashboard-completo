VERSÃO v36.5 - CORREÇÃO OBSERVATIONS / EM TRATATIVA

Correção aplicada para o caso identificado na BSP 26-030:
- Quando a coluna OBSERVATIONS possuir "Aguardando liberação para envio", a BSP entra em "Em tratativa".
- A informação passa a aparecer no duplo clique/detalhamento da BSP.
- A regra continua ativa enquanto a observação existir na planilha/API.
- Quando a observação for removida, a BSP volta a seguir o status normal calculado pelo sistema.

Regras consideradas como tratativa:
- Revisão de P.O
- Aguardando liberação para envio
- Entrega parcial

Ajustes técnicos:
1. netlify/functions/projects.js
   - Criado leitor robusto de observações para variações de coluna:
     OBSERVATIONS, OBSERVATION, Obs, Observação, Notes, Comments etc.
   - Corrigida deduplicação de Tags/ISOs para não perder observações quando existir linha duplicada.
   - As observações de linhas duplicadas agora são mescladas antes do cálculo da tratativa.
   - O projeto é redecorado com a regra de tratativa após o rollup final dos spools.

2. site/app.js
   - O painel do cliente agora também lê matches de tratativa já enviados pela API.
   - O status, filtro e modal de duplo clique reconhecem a observação crítica mesmo quando ela veio de uma Tag/ISO.
   - Removida duplicidade visual de motivos iguais.

Validações executadas:
- node --check netlify/functions/projects.js
- node --check site/app.js
- npm run lint
- Simulação local com BSP 26-030 contendo "Aguardando liberação para envio" em OBSERVATIONS de uma Tag/ISO duplicada.

Resultado esperado:
- BSP 26-030 aparece como "Em tratativa".
- Ao dar duplo clique, aparece o aviso com "Aguardando liberação para envio".
