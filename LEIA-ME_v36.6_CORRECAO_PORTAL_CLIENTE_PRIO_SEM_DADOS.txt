Versão v36.6 - Correção Portal do Cliente sem dados

Correção principal:
- Corrigido o filtro do Portal do Cliente quando o usuário cliente foi criado sem clientKey/clientName.
- Antes, o cabeçalho podia mostrar "PRIO" pelo nome do usuário, mas o backend filtrava apenas clientKey/clientName. Resultado: painel do cliente carregava sem BSPs, cards ficavam em "--" e não apareciam unidades/BSPs.
- Agora, se clientKey/clientName estiverem vazios, o backend usa também name/username do usuário cliente como escopo compatível, com comparação segura por tokens não genéricos.

Melhorias adicionadas:
- Portal do Cliente agora mostra mensagem de carregamento/erro visível no próprio painel, evitando ficar parado em "--" quando a API estiver carregando ou falhar.
- Cache local do frontend atualizado para v5, evitando reaproveitar cache antigo vazio.
- Service Worker atualizado para v36.6 e app.js/app.css versionados no index para forçar o navegador/PWA a carregar os arquivos novos.

Regras anteriores mantidas:
- ON HOLD aparece no duplo clique somente quando a BSP estiver em On Hold.
- Observações críticas mantêm BSP em "Em tratativa":
  * Revisão de P.O
  * Aguardando liberação para envio
  * Entrega parcial
- Observações aparecem no detalhamento da BSP/Tag.

Validações locais:
- node --check site/app.js
- node --check netlify/functions/projects.js
- npm run lint
- Teste de API local simulando usuário PRIO sem clientKey/clientName retornou 86 projetos PRIO pelo snapshot do ZIP.

Observação:
- npm run validate apontou ausência de SUPABASE_URL e SUPABASE_SERVICE_ROLE_KEY no ambiente local de teste. Isso é configuração local do ambiente, não erro da correção.
